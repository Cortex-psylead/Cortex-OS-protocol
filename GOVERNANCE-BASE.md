# 🏛️ Base Governance Framework: Cortex OS Protocol

This document defines the unalterable principles governing the relationship between the **Master Agent**, the **Ethical/Health Committees**, and **User Sovereignty**. Cortex OS does not aim to tutor the individual, but to provide a technical and clinical infrastructure for consciousness.

---

## ⚖️ Fundamental Principles

### 1. Primacy of Will (User Sovereignty)
The user is the sole sovereign of their device and their attention.
* **The "Kill Switch" (Overrule):** No clinical restriction or suggestion can override a direct, conscious order from the user.
* **Right to Error:** The system guarantees the user’s right to choose extreme performance or prolonged use, even if it contradicts the health recommendations provided by the committees.

### 2. Non-Invasive Intervention (Nudge Architecture)
Cortex OS operates under the concept of **Informational Friction** instead of authoritarian blocking.
* **Warning, Not Prohibition:** If the system detects a physiological risk (visual fatigue, high stress, lack of sleep), the Agent will subtly inform and increase system resistance (e.g., minimal latencies or gradual light shifts).
* **Dynamic Consent:** The user can ignore or override any protection with a single command. The system offers the roadmap to health, but never forces the user to follow the path.

### 3. Committee Transparency
The limits established by the committees (Psychology, Neuroscience, Ethics, and Engineering) must be:
* **Auditable:** Any user can consult which rules are active and the scientific evidence they are based on.
* **Neutral:** It is strictly forbidden to implement rules that favor commercial interests or hide hardware functions (NPU/GPU) from the user.

---

## 🛡️ Intervention Levels (User-Defined)

The user defines the extent of the system’s authority:

1. **Level 0 (Absolute Sovereignty):** The system acts only as a passive observer. It provides data but never intervenes with the hardware.
2. **Level 1 (Suggested Support):** The Agent applies mild automatic adjustments (light filters, binaural audio) that the user can easily revert.
3. **Level 2 (Clinical Commitment):** The user voluntarily delegates authority to the Agent for critical situations (e.g., sensory crises or anxiety attacks), allowing for pre-agreed stimulus blocking.

---

## 🧩 Committee Responsibilities

Committees will inherit this protocol with the mission to expand safety margins in areas such as:
* **Sensory Safety:** Light frequency and decibel limits for neurodivergence.
* **Metabolic Integrity:** Suggestions based on circadian rhythms for night workers and students.
* **Clinical Privacy:** Absolute shielding of local biometric data.
